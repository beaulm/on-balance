- **Hardening landed (#49 → #74)** — Per-fingerprint rate limiting (10/hour), origin checking, timestamp drift validation, standardized error codes, and global eviction for the in-memory rate-limit map. February's bet on hardening before user exposure paid off.
- **Deploy preview edge cases caught and fixed** — 7 follow-up commits on the visualization branch addressed same-origin checks for deploy previews, malformed file handling, partial response caching, and module slug change state syncing. Real deployment surfaced issues that local dev couldn't.

### Process Improvements

- **CSS Highlight API with fallback pattern** — Three-tier opacity system (`0.1` / `0.2` / `0.3`) keeps WCAG AA contrast while providing visible differentiation. Progressive enhancement approach means the feature degrades gracefully.

## What Didn't Work?

### Failed Experiments

- **Heartbeat cadence dropped again** — Only 2 heartbeats filed (March 2 and 9), with none in the back half of the month. Same pattern as the February gap. The March 2 heartbeat noted the project was intentionally paused for internal processing, but the cadence never resumed after #74 merged on March 9.
- **~2.5 week gap between feature PRs** — #74 merged March 9, #76 work didn't start until March 25. Velocity was concentrated in bookend bursts rather than steady flow.

### Lessons Learned

- **Deploy previews are a real testing environment** — The 7 follow-up commits on the visualization branch show that Netlify deploy previews surfaced origin gate, CORS, and caching issues that localhost didn't. Worth treating deploy preview testing as a deliberate step, not just a side effect.
- **Same-origin checks need deploy preview awareness** — Both `record-resonance` and `get-resonance` needed fixes to allow deploy preview origins (`deploy-preview-*--on-balance.netlify.app`), suggesting origin validation should be factored into a shared utility.
