## What Didn't Work?

### Failed Experiments

- **Auto-focus on popup** - Had to remove because it clears browser text selection, preventing copy/paste
- Initial element node offset calculation was overly complex before simplifying to Range API

### Lessons Learned

- **Preserve user affordances** - Text selection is a core browser feature; don't break it for UX additions
- **Range API is powerful** - `range.toString().length` handles both text and element nodes cleanly
- **Right-click needs special handling** - Context menus require `e.button !== 0` checks
